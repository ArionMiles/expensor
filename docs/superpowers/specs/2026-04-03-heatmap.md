# Group E — Spending Heatmap: Design Spec

**Date:** 2026-04-03  
**Features:** Heatmap spending chart (#11)  
**Scope:** Two new backend queries, one new API endpoint, two SVG-based React widgets on the Dashboard

---

## Overview

Two independent heatmap widgets are added to the Dashboard:

1. **Hour-of-day grid** — a 7 × 24 grid (day-of-week rows × hour-of-day columns) showing _when during the week_ you transact. This is the closest analogue to GitHub's contribution graph, but for spending time patterns.

2. **Day-of-month strip** — a 1 × 31 strip (days 1–31) showing _which calendar days_ you spend most.

Both widgets share:
- A toggle button: **Amount** | **Count** — switches the intensity metric without refetching.
- A colour scale legend (5 buckets: empty → low → medium → high → very high).
- A hover tooltip showing the exact value.

## Backend

### New store methods

**File:** `backend/internal/store/store.go`

```go
type WeekdayHourBucket struct {
    Weekday int     `json:"weekday"` // 0=Sunday … 6=Saturday (PostgreSQL DOW)
    Hour    int     `json:"hour"`    // 0–23
    Amount  float64 `json:"amount"`
    Count   int     `json:"count"`
}

type DayOfMonthBucket struct {
    Day    int     `json:"day"`    // 1–31
    Amount float64 `json:"amount"`
    Count  int     `json:"count"`
}

// GetSpendingHeatmap returns both datasets in one call.
func (s *Store) GetSpendingHeatmap(ctx context.Context) (*HeatmapData, error)
```

```go
type HeatmapData struct {
    ByWeekdayHour []WeekdayHourBucket `json:"by_weekday_hour"`
    ByDayOfMonth  []DayOfMonthBucket  `json:"by_day_of_month"`
}
```

**SQL:**

```sql
-- Weekday × hour grid
SELECT
    EXTRACT(DOW  FROM timestamp)::int AS weekday,
    EXTRACT(HOUR FROM timestamp)::int AS hour,
    COALESCE(SUM(amount), 0)          AS amount,
    COUNT(*)                          AS count
FROM transactions
GROUP BY weekday, hour
ORDER BY weekday, hour;

-- Day of month strip
SELECT
    EXTRACT(DAY FROM timestamp)::int AS day,
    COALESCE(SUM(amount), 0)         AS amount,
    COUNT(*)                         AS count
FROM transactions
GROUP BY day
ORDER BY day;
```

Both queries run across all-time data (no date filter). The frontend's toggle switches the display metric client-side without a new fetch.

### New API endpoint

**File:** `backend/internal/api/handlers.go`

```go
// HandleGetHeatmap handles GET /api/stats/heatmap
func (h *Handlers) HandleGetHeatmap(w http.ResponseWriter, r *http.Request) {
    if h.store == nil {
        writeError(w, http.StatusServiceUnavailable, "database not connected")
        return
    }
    data, err := h.store.GetSpendingHeatmap(r.Context())
    if err != nil {
        h.logger.Error("get heatmap", "error", err)
        writeError(w, http.StatusInternalServerError, "failed to fetch heatmap data")
        return
    }
    writeJSON(w, http.StatusOK, data)
}
```

Route: `GET /api/stats/heatmap` — registered in `server.go` alongside the existing `GET /api/stats/charts`.

## Frontend

### Data fetching

**File:** `frontend/src/api/queries.ts`

```ts
export function useHeatmapData() {
  return useQuery({
    queryKey: ['heatmap'],
    queryFn: () => client.get<HeatmapData>('/api/stats/heatmap'),
    staleTime: 5 * 60 * 1000, // 5 minutes
  })
}
```

### Widget 1: `WeekdayHourHeatmap`

**File:** `frontend/src/components/WeekdayHourHeatmap.tsx`

**Layout:**
```
         00 01 02 03 … 22 23
   Sun  [ ][ ][ ][ ]…[ ][ ]
   Mon  [ ][ ][ ][ ]…[ ][ ]
   …
   Sat  [ ][ ][ ][ ]…[ ][ ]
```

- 7 rows × 24 columns = 168 cells.
- Each cell is an SVG `<rect>` with `width=14 height=14 rx=2`.
- Row labels (Sun–Sat) on the left; column labels (0, 6, 12, 18) at the top (not every hour — too crowded).
- Cell colour: linear interpolation from `--muted` (zero) to `--primary` (max) using 5 discrete steps.

**Colour scale:**
```ts
function intensityColor(value: number, max: number): string {
  if (max === 0 || value === 0) return 'hsl(var(--muted))'
  const ratio = Math.min(value / max, 1)
  const step = Math.ceil(ratio * 4) // 1–4
  const opacities = [0.2, 0.4, 0.65, 0.9]
  return `hsla(var(--primary-raw), ${opacities[step - 1]})`
}
```

`--primary-raw` is the raw H S L values without `hsl()` wrapper, added to the CSS variable set.

**Tooltip:** On `mouseenter`, a floating `<div>` (absolutely positioned) shows:
```
Mon 14:00
₹2,340  •  3 transactions
```

**Toggle:** Amount / Count buttons above the grid. The `metric: 'amount' | 'count'` state is local to the component; no refetch required.

### Widget 2: `DayOfMonthHeatmap`

**File:** `frontend/src/components/DayOfMonthHeatmap.tsx`

**Layout:**
```
 1  2  3  4  5 … 29 30 31
[ ][ ][ ][ ][ ]…[ ][ ][ ]
```

- Single row of 31 cells, each `20 × 40px` (taller than wide to give room for the day number).
- Day number rendered as `<text>` inside each SVG `<rect>`.
- Same colour scale as Widget 1 (shared utility function).
- Cells for days that don't exist in the current month (e.g. day 31 in a 30-day month) are shown at 20% opacity with no tooltip — the data simply won't have a bucket for them.

**Tooltip:** Same pattern as Widget 1.

### Dashboard integration

**File:** `frontend/src/pages/Dashboard.tsx`

Both widgets placed in a new "Spending Patterns" section below the existing chart row. The section has a shared metric toggle (Amount / Count) that controls both widgets simultaneously via lifted state.

```tsx
const [metric, setMetric] = useState<'amount' | 'count'>('amount')
// ...
<section>
  <header>
    Spending Patterns
    <MetricToggle value={metric} onChange={setMetric} />
  </header>
  <WeekdayHourHeatmap data={heatmap.by_weekday_hour} metric={metric} />
  <DayOfMonthHeatmap  data={heatmap.by_day_of_month}  metric={metric} />
</section>
```

### Loading / empty states

- While `useHeatmapData` is loading: a skeleton placeholder matching the grid dimensions.
- If all values are zero (no transactions yet): a muted overlay with "No transaction data yet".

```

```
